# 📦 WispByte 部署指南

## 项目结构
```
finance_news/
├── main.py              # FastAPI 主程序
├── requirements.txt     # Python 依赖
└── static/
    ├── index.html       # 前端页面
    ├── style.css        # 样式文件
    └── app.js           # JavaScript 交互
```

---

## 🚀 部署步骤

### 步骤 1：登录 WispByte
1. 访问 https://wispbyte.com/
2. 使用 Google/GitHub/邮箱登录

### 步骤 2：创建服务
1. 点击 Dashboard 中的 **"Create Server"**
2. 填写服务名称（如：finance-news）
3. 服务描述（可选）
4. **服务类型选择：Free Plan**（免费套餐）
5. **运行环境选择：Python**
6. 点击 **"Create Server"** 创建

### 步骤 3：配置环境
1. 创建完成后点击 **"MANAGE SERVER"** 进入管理页面
2. 选择左侧 **"Startup"** 选项
3. 配置以下内容：
   - **Python 版本**: 选择 `python:3.11` 或更高版本
   - **添加依赖包**: 
     - 输入 `fastapi` 点击 Add Package
     - 输入 `uvicorn` 点击 Add Package
     - 输入 `requests` 点击 Add Package
     - 点击 **Save Package** 保存
   - **入口文件**: 输入 `main.py` 并保存

### 步骤 4：上传项目文件
1. 点击左侧 **"Files"** 进入文件管理
2. 上传以下文件：
   - 上传 `main.py`
   - 上传 `requirements.txt`
   - 创建 `static` 文件夹
   - 在 `static` 文件夹中上传：
     - `index.html`
     - `style.css`
     - `app.js`

**上传方法**:
- 可以直接拖拽文件上传
- 或点击文件夹后使用上传按钮

### 步骤 5：配置端口
1. 点击左侧 **"Console"** 查看控制台
2. 记录分配的端口号（如：8080）
3. 在文件管理中编辑 `main.py`
4. 确保端口配置正确：
   ```python
   port = int(os.environ.get("PORT", 8080))  # 使用 WispByte 分配的端口
   ```

### 步骤 6：启动服务
1. 在 **Console** 页面
2. 点击 **"Start"** 按钮启动服务
3. 观察控制台输出，看到以下信息表示启动成功：
   ```
   INFO:     Application startup complete.
   INFO:     Uvicorn running on http://0.0.0.0:8080
   ```

### 步骤 7：访问服务
1. 在 **Console** 页面查看分配的公网 IP 和端口
2. 格式如：`http://xxx.xxx.xxx.xxx:8080`
3. 在浏览器中打开该地址即可访问

## ⚠️ 注意事项

1. **端口**: 必须使用 WispByte 分配的端口，在 Console 页面查看
2. **内存限制**: 512MB，避免大量数据处理
3. **存储**: 1G 空间，不要存储大量文件
4. **网络**: 确保新浪财经 API 可访问
5. **自动刷新**: 前端每 60 秒自动刷新一次新闻

## 🔧 常见问题

### Q: 服务启动失败？
A: 检查以下几点：
- 入口文件是否设置为 `main.py`
- 依赖包是否安装完成
- 端口配置是否正确
- 查看 Console 中的错误日志

### Q: 无法访问？
A: 确认：
- 服务是否已启动
- 端口号是否正确
- 防火墙是否阻止访问

### Q: 新闻无法加载？
A: 可能是：
- 网络问题，稍后重试
- API 接口变化，需要更新代码
- 触发了反爬机制，增加延时
